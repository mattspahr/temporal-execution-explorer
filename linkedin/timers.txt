I built an interactive demo to visualize how Temporal handles durable timers.

It walks through a checkout workflow that hits a 24-hour fraud review hold after charging the card. The worker goes completely idle — no compute, no polling, no cron job holding state. Temporal Server holds the timer durably. When it fires, a worker picks back up and finishes the workflow.

What I like about building it this way:
  - You can watch the 24-hour timer fast-forward with a live clock counting up
  - The worker status shows it going idle during the wait — no resources consumed
  - Switch between TypeScript, Go, Python, and Java to see how each SDK expresses sleep/timer

There aren't any workflows running. This is purely a frontend visualization of the mechanics.
